# 系统设置功能说明

## 概述

系统设置页面提供了对所有配置的集中管理，按类别分为以下几个部分：

1. 🕷️ 爬虫配置
2. 🤖 模型配置
3. 🗄️ Redis配置
4. 📧 邮件配置
5. ⏰ 定时任务配置

📖 **界面使用指南**: [SYSTEM_SETTINGS_UI_GUIDE.md](./SYSTEM_SETTINGS_UI_GUIDE.md)

## 配置类别详解

### 1. 爬虫配置 (Crawler Configuration)

用于配置网站爬取的相关参数。

**配置项：**
- **Sitemap URL**: 站点地图地址（必填）
- **超时时间**: HTTP请求超时时间（秒）
- **最大URL数量**: 限制爬取的URL数量（留空表示无限制）
- **请求间隔**: 两次请求之间的延迟时间（秒）
- **输出目录**: 爬取结果的保存路径
- **User Agent**: 爬虫的用户代理字符串
- **URL包含规则**: 只爬取包含指定模式的URL
- **URL排除规则**: 排除包含指定模式的URL

**保存方式：** 保存到 `app/config/crawler_config.json` 文件
**生效方式：** 立即生效，无需重启服务

---

### 2. 模型配置 (LLM Configuration)

用于配置大语言模型和Embedding模型的参数。

**配置项：**
- **API Key**: OpenAI API密钥（敏感信息）
- **API Base URL**: API基础URL（例如: https://api.openai.com/v1）
- **模型名称**: 主模型名称（例如: gpt-4o-mini）
- **Embedding模型**: 向量嵌入模型名称（例如: text-embedding-3-small）
- **Embedding维度**: 向量维度（默认: 1536）
- **最大输入Token**: 模型最大输入token数（可选）
- **最大输出Token**: 模型最大输出token数（可选）

**保存方式：** 保存到 `.env` 文件
**生效方式：** 需要重启服务才能生效

**注意事项：**
- API Key等敏感信息仅保存在服务器端，不会暴露给前端
- 修改配置后必须重启服务才能使新配置生效
- 如果某些配置项留空，将使用默认值

---

### 3. Redis配置 (Redis Configuration)

用于配置Redis向量数据库的连接信息。

**配置项：**
- **Redis URL**: Redis连接地址（例如: redis://localhost:6379）

**保存方式：** 保存到 `.env` 文件
**生效方式：** 需要重启服务才能生效

**注意事项：**
- Redis用于存储面试题的向量表示，支持语义搜索
- 如果使用Docker部署，Redis URL应为 `redis://redis:6379`

---

### 4. 邮件配置 (Email Configuration)

用于配置SMTP邮件发送功能。

**配置项：**
- **SMTP服务器**: SMTP服务器地址（例如: smtp.qq.com）
- **SMTP端口**: SMTP服务器端口（例如: 587）
- **用户名**: SMTP登录用户名（通常是邮箱地址）
- **密码**: SMTP密码或授权码
- **测试邮箱**: 用于测试邮件发送的邮箱地址

**保存方式：** 保存到 `.env` 文件
**生效方式：** 需要重启服务才能生效

**功能：**
- **保存配置**: 保存邮件配置到.env文件
- **测试邮件发送**: 发送测试邮件到指定的测试邮箱，验证配置是否正确

**注意事项：**
- QQ邮箱需要使用授权码，而不是登录密码
- Gmail需要开启"不够安全的应用访问"或使用应用专用密码
- 测试邮件会发送一个示例面试题到测试邮箱

---

### 5. 定时任务配置 (Scheduler Configuration)

用于配置自动爬取任务的执行时间。

**配置项：**
- **执行时间**: 每天执行爬取任务的时间（时:分）

**保存方式：** 保存到 `.env` 文件（SCHEDULER_HOUR 和 SCHEDULER_MINUTE）
**生效方式：** 需要重启服务才能生效

**注意事项：**
- 默认执行时间为每天 22:00
- 定时任务会自动爬取配置的Sitemap并生成面试题

---

## 使用流程

### 首次配置

1. 进入系统设置页面
2. 按顺序配置各个类别：
   - 先配置爬虫设置（Sitemap URL是必需的）
   - 配置模型设置（API Key是必需的）
   - 配置Redis设置（如果需要向量搜索功能）
   - 配置邮件设置（如果需要邮件通知功能）
   - 配置定时任务（可选）
3. 每个配置类别点击"保存配置"按钮
4. 对于需要重启的配置（模型、Redis、邮件、定时任务），重启服务使配置生效

### 修改配置

1. 切换到对应的配置标签页
2. 修改配置项
3. 点击"保存配置"按钮
4. 如需重启，重启服务

### 重置配置

在每个配置页面都可以点击"重置"按钮，重新从服务器加载最新配置。

## API接口说明

### 获取系统配置
```
GET /api/system-config
```
返回所有配置项，按类别分组。

### 更新爬虫配置
```
PUT /api/config
Body: { sitemap_url, timeout, max_urls, ... }
```

### 更新模型配置
```
PUT /api/llm-config
Body: { openai_api_key, openai_api_base, openai_model, ... }
```

### 更新Redis配置
```
PUT /api/redis-config
Body: { redis_url }
```

### 更新邮件配置
```
PUT /api/email-config
Body: { smtp_server, smtp_port, smtp_user, smtp_password, smtp_test_user }
```

### 更新定时任务配置
```
PUT /api/scheduler-config?hour=22&minute=0
```

### 测试邮件发送
```
POST /api/test-email
```
发送测试邮件到配置的测试邮箱。

## 常见问题

### Q: 为什么修改配置后需要重启服务？

A: 模型、Redis、邮件和定时任务配置在服务启动时加载到内存中。修改这些配置后，需要重启服务才能重新加载配置。

### Q: 爬虫配置为什么不需要重启？

A: 爬虫配置每次使用时都会从配置文件读取，所以修改后立即生效。

### Q: API Key安全吗？

A: 是的。API Key等敏感信息：
- 仅保存在服务器的.env文件中
- 前端显示时会进行脱敏处理
- 不会通过API明文传输

### Q: 如何验证邮件配置是否正确？

A: 在邮件配置页面填写完配置后，点击"测试邮件发送"按钮。系统会发送一封测试邮件到配置的测试邮箱。

### Q: 配置保存在哪里？

A: 
- 爬虫配置：`app/config/crawler_config.json`
- 其他配置：`.env` 文件

## 技术实现

### 后端
- 使用 FastAPI 提供 RESTful API
- 使用 python-dotenv 管理环境变量
- 配置文件持久化到文件系统

### 前端
- 使用 Vue 3 + Ant Design Vue
- 按类别分标签页展示配置
- 提供保存、重置、测试等功能
- 实时反馈操作结果

---

## 最佳实践

1. **定期备份配置**: 特别是.env文件，包含所有敏感信息
2. **使用环境变量**: 在生产环境中，建议使用环境变量而非硬编码配置
3. **测试后再应用**: 修改重要配置前，先在测试环境验证
4. **最小权限原则**: API Key等凭证应使用最小必要权限
5. **监控配置变更**: 记录配置修改历史，便于问题排查
